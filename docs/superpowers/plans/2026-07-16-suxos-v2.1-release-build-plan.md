# SuxOS v2.1 Release — Build Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Ownership tags: **[operator]** = interactive Claude session, **[pipeline]** = the three-loop autonomy (seed via `dispatch`), **[human]** = owner-only action (flag with a 🔴 NEEDS YOU banner).

**Goal:** Ship **SuxOS v2.1** — the v2 arc reconciled to *done* and certified green: every vX-ledger row landed at origin/main, the eligible backlog drained to zero, all autonomy loops re-enabled (fixer last, per .github#282), the spine green, a release record + GitHub Release `suxos-v2.1` cut on `SuxOS/.github`, and the §4 seven-day gate armed so the next arc gets picked by measurement.

**What "v2.1" means:** `docs/design/2026-07-16-suxos-vx-next-arc.md` defines the v2 arc (product op-engine + fabric stability) and found it ~90% landed but frayed. **v2.1 is the completion increment of that arc** — not a package version. Package semver (`sux` 0.4.0, `suxlib` 0.0.0…) is untouched; the release object lives on `.github`, the org's system repo, tagged `suxos-v2.1`.

**Shape:** Three phases mirroring the operator's standing commands: **Land** (finish the ledger remainder — mostly pipeline drain), **Audit-until-green** (a bounded verification loop over the ledger + spine), **Release + burn-in** (fixer re-enable → release cut → 7-day gate). Most build work is *pipeline-executed*; the operator unjams, verifies, and cuts the release; two actions are human-gated.

**Surfaces:** `gh` CLI (issues/PRs/workflows/releases), git against verified SHAs, the three-loop pipeline in `SuxOS/.github`, the fabric health spine (Grafana `/d/suxos-fabric-health`, `suxos_*` metrics), `~/.claude/fabric.json`.

## Global Constraints

- **Verify provider health before mutating state.** GitHub was actively degraded on 2026-07-16 (Actions API 503s; a fetch *rewound* `origin/main` from a stale replica). No close-stale/reconcile automation and no audit verdicts while Task 0's health gate fails.
- **Audit against verified SHAs.** Pin to the most advanced known-merged SHA; a fresh fetch during an incident can be worse than a stale one. Local checkouts still lie (vX invariant 4).
- **Every `disabled_manually` carries a tracking issue + expiry condition** (vX invariant 1). Target end-state: zero untracked disables.
- **Docs must not lead reality** (vX invariant 2). The two-vault model documents what production *does*, not what a parked sweep wished.
- **Retire = de-register + archive** (vX invariant 3).
- **Pipeline discipline:** never fabricate issues to feed the drain; never hand-force-push `suxbot[bot]` App branches; unjam via the documented playbooks (disabled-required-check, onboarding-window, path-filtered-required-check, five-hour-limit false-red, author_association starvation).
- **Tier A needs an explicit human yes:** repo archive, branch deletion of a deliberately-parked branch, force-push, anything irreversible.
- **`.github` main is the live pipeline; `sux` main auto-deploys.** Everything lands via PR; `Self automerge` takes green PRs.
- **Right-size model spend** per `docs/design/budget-and-cadence.md`; the drain is throughput-bound — do not add cadence.
